# Contributing Feedback

Thank you for helping us improve! Because this is a public feedback portal and our core execution backlog is private, we have a specific workflow for handling your submissions.

## How to report bugs

1. Check existing issues to ensure your bug hasn't already been reported.
2. Use the Bug Report issue form. 
3. Fill in the `Product` field so maintainers can route the issue to the right product.
4. Provide exact steps to reproduce. If the issue is related to a specific browser extension environment or cloud deployment, include the exact versions.
5. Issues submitted without clear reproduction steps will be labeled `status: needs-repro` and eventually closed if inactive.

## How feature requests are evaluated

We want to hear your ideas. For MVP, we prefer that you start all feature requests in **Discussions -> Ideas**. 

If you already know the product, include it in the `Product` field on the feature request form. We apply the matching product label automatically during triage.

If a feature aligns with our roadmap, a maintainer will convert the Discussion into a tracked Issue and label it `status: accepted`. Please understand that we cannot build every request, and some ideas may be marked as "not planned" to keep our backlog realistic.
