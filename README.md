# Public Feedback Portal

This repository is the public feedback hub for our application. We use this space to track reproducible bugs, gather feature requests, and answer community questions.

## Where to go

To keep our triage process fast and effective, please route your request to the correct location:

* **Have a question or need help?** → [Start a Discussion in Q&A](../../discussions/categories/q-a)
* **Have a product idea?** → [Start a Discussion in Ideas](../../discussions/categories/ideas)
* **Found a reproducible bug?** → [Open a Bug Issue](../../issues/new?assignees=&labels=status%3A+needs-triage&projects=&template=bug_report.yml&title=%5BBUG%5D+)
* **Need private account support?** → Contact support@example.com
* **Reporting a security issue?** → Read our [SECURITY.md](SECURITY.md)

When you file a bug or feature request, fill out the `Product` field in the form. We use that to apply the right product label during triage, so you do not need to manage labels yourself.

## Expectations

1. **Public Repository:** Everything posted here is visible to the public. Do not post API keys, passwords, extension tokens, or sensitive user data.
2. **Triage Timelines:** We aim to review all new issues within 3 business days and provide a triage decision within 7 business days.
3. **Respectful Participation:** Keep conversations constructive. We reserve the right to close issues or discussions that violate standard community guidelines.
