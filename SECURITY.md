# Security Policy

The security of our users is our top priority. **Do not report security vulnerabilities in public issues or discussions.**

## Supported Versions

We currently provide security updates for the latest stable release of our software.

## Reporting a Vulnerability

If you discover a security vulnerability, please report it privately to our security team. 

1. Email: security@example.com
2. Include a description of the vulnerability and exact steps to reproduce it.
3. Do not paste sensitive logs, API keys, or active session tokens into standard emails. If logs are required, we will provide a secure upload link.

You should receive a response within 48 hours acknowledging receipt of the vulnerability. We will provide updates as we investigate and deploy a fix.
